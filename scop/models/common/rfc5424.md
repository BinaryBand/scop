# RFC 5424 Model (`rfc5424.py`)

Pydantic model representing the SCOP-relevant slice of
[RFC 5424: The Syslog Protocol](https://www.rfc-editor.org/info/rfc5424)
(Gerhards, R., March 2009, Standards Track).

## Purpose

`RFC5424` is a `BaseRFC` subclass that renders `RFC5424.md` at the
repository root using the shared `RFC7841.md.j2` template. It documents
the RFC 5424 fields that SCOP builds on — PRI/Severity, HEADER fields,
STRUCTURED-DATA, and MSG — in the same RFC 7841 boilerplate format as
the SCOP specification itself.

## Data source

`static/META5424.yaml` — provides `title`, `abstract`, `introduction`,
and `sections` (5 sections: Message Format, PRI, HEADER Fields,
Structured Data, MSG).

## Key overrides from `BaseRFC`

| Field                 | Value                      | Reason                                 |
| --------------------- | -------------------------- | -------------------------------------- |
| `conformance_partial` | `""`                       | RFC 5424 has no SCOP conformance block |
| `terms`               | RFC 5424 field definitions | Replaces SCOP-specific defaults        |
| `principles`          | RFC 5424 design goals      | Replaces SCOP-specific defaults        |

## Relationship to SCOP

SCOP uses RFC 5424 as its event envelope. Specifically:

- `pri` → severity-only (0–7); facility fixed at 0
- `ts`, `app`, `pid` → optional HEADER fields carried through unchanged
- `msgid` → repurposed as the SCOP event type identifier
- `msg` → maps to RFC 5424 MSG; MUST be a complete human-readable line
- STRUCTURED-DATA → not used by SCOP (structured content lives in JSON fields)
